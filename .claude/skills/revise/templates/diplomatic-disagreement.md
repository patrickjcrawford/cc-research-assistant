# Diplomatic Disagreement Phrasing

When the user decides to push back on a referee comment (DISAGREE classification), use these patterns:

## Acknowledge first
"We appreciate the referee's careful attention to [topic]."
"This is an important point that we addressed by [action]."

## Explain the constraint
"We considered [referee's suggestion] but found that [specific reason] makes this approach less suitable for our setting."
"[Referee's approach] would require [specific assumption] that is unlikely to hold in our context because [reason]."

## Show you tried
"We implemented [referee's suggestion] as a robustness check (see Appendix Table X) and find [result]. Our preferred specification remains [approach] because [reason]."

## Never say
- "The referee is wrong"
- "We disagree" (without explanation)
- "This is well known" (even if it is)
- "We already addressed this" (show where, don't just assert)

## The structure
1. Thank / acknowledge
2. Clarify what you did
3. If disagreeing: show evidence or cite authority
4. If partially agreeing: show the change AND explain what you kept
