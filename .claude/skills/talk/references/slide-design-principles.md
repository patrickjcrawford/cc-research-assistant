# Slide Design Principles

## One Idea Per Slide
Every slide has exactly one message. If you need two bullets to explain the slide's point, it's two slides.

## Figures Over Tables
Tables on slides are hard to read. Convert key results to figures. If you must show a table, highlight the one number that matters.

## Minimal Text
- Max 4-5 lines per slide
- No full sentences (except quotes)
- Audience reads faster than you speak -- less text means they listen

## Font Sizes
- Title: >=24pt
- Body: >=18pt
- Footnotes/sources: >=12pt
- If you can't read it from the back row, it's too small

## Colors
- Use color purposefully (highlight treatment group, mark significance)
- Red/green color blindness affects ~8% of men -- use blue/orange instead
- Define a `result` color for highlighted findings and use it consistently

## Equations
- Show the equation, but explain it in words
- Highlight the key parameter (use `\color{}` or `\boxed{}`)
- Max 2 equations per slide

## Builds
- Use `\pause` for sequential reveals when the logical flow requires it
- Don't over-animate -- most slides should appear complete
- Use `\only<>` for equation buildup in structural model presentations

## Tables on Slides
- Max 4-5 columns (fewer than the paper)
- Highlight the key coefficient with color
- Gray out control rows
- Full table belongs in backup slides

## Visual Rhythm
- Alternate dense slides (data, results) with sparse slides (key finding, transition)
- Never three dense slides in a row
- Transition slides between major sections: a single sentence or question

## The 3-Second Test
Can you tell what the slide is about within 3 seconds? If not, simplify.

## Citations
- Author-year on slides (e.g., "Card (1994)")
- Full bibliographic details in backup slides only
- Never clutter a result slide with citation details

## Speaker Notes
- Every slide should have notes with: the key point, timing, anticipated questions
- Note which backup slide answers each likely question
