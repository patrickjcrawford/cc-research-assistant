# Working Paper Format Standard

All papers in this project are authored in Quarto (`.qmd`). The primary file is `paper/main.qmd`. PDF output is produced via `quarto render` using XeLaTeX under the hood. This rule applies to the writer, writer-critic, and verifier agents.

---

## Compilation

```bash
# Render paper to PDF
quarto render paper/main.qmd

# Live preview (auto-reloads on save)
quarto preview paper/main.qmd
```

Quarto manages all LaTeX passes internally. Do not run `latexmk` or `xelatex` directly.

---

## Document Settings

Formatting is controlled by `_quarto.yml` at the project root. Key settings already configured:

```yaml
format:
  pdf:
    documentclass: article
    fontsize: 10pt
    geometry: margin=1in
    pdf-engine: xelatex
    indent: true
    number-sections: true
    toc: false
```

Double-spacing is **always on** for working papers and submissions. It is included in `_quarto.yml` by default:

```yaml
format:
  pdf:
    include-in-header:
      - text: |
          \usepackage{setspace}
          \doublespacing
```

To temporarily disable (e.g., for a department seminar handout), override in the individual `.qmd` YAML with `\singlespacing`.

## Citations

Citations use Quarto citeproc + CSL, configured in `_quarto.yml`:

```yaml
citeproc: true
bibliography: references.bib
csl: https://www.zotero.org/styles/apa
```

Cite in text with `@authorYear3keywordsTitle` (inline) or `[@authorYear3keywordsTitle]` (parenthetical). BibTeX key format: `authorYear3keywordsTitle` (e.g., `smith2024effectMinimumWage`). Do not add `biblatex` or `natbib` packages manually.

---

## Cross-References

Use Quarto's native cross-reference syntax. Do not use `\ref{}`, `\cref{}`, or load `hyperref`/`cleveref` manually â€” Quarto injects these for PDF output automatically.

| Target | Label syntax | Reference syntax |
|--------|-------------|-----------------|
| Figure | `#| label: fig-name` | `@fig-name` |
| Table | `#| label: tbl-name` | `@tbl-name` |
| Section | `## Title {#sec-name}` | `@sec-name` |
| Equation | `$$ ... $$ {#eq-name}` | `@eq-name` |

## Tables

### From R/Python/Julia scripts

Scripts export bare `tabular` environments (no float wrapper). Include them in `main.qmd` via a raw LaTeX block:

````markdown
```{=latex}
\begin{table}[htbp]
\centering
\begin{threeparttable}
\caption{Effect of X on Y}\label{tab:main}
\input{paper/tables/estimation/reg_main_specification.tex}
\begin{tablenotes}\small
  \item \textit{Notes:} Robust standard errors in parentheses.
  * p < 0.10, ** p < 0.05, *** p < 0.01
\end{tablenotes}
\end{threeparttable}
\end{table}
```
````

### Hand-written tables

Write directly in `.qmd` using a raw LaTeX block with `tabularray`:

````markdown
```{=latex}
\begin{talltblr}[
  caption = {Effect of X on Y},
  label   = {tab:main},
  note{*} = {Robust standard errors in parentheses.},
]{colspec = {lcccc}, rowsep = 4pt}
\toprule
  & (1) & (2) & (3) & (4) \\
\midrule
...
\bottomrule
\end{talltblr}
```
````

### Rules (apply to both)

- `\toprule`, `\midrule`, `\bottomrule` (booktabs) â€” never `\hline`
- No vertical rules
- Every table has notes (via `\begin{tablenotes}` or tabularray's `note{}`)

## Figures

Figures are generated by R/Python/Julia scripts and saved to `paper/figures/`. Include them via standard Quarto chunk options:

````markdown
```{r}
#| label: fig-main-results
#| fig-cap: "Effect of X on Y. Notes: Sample is ..."
#| fig-width: 7.5
#| fig-height: 4

knitr::include_graphics("paper/figures/01_analysis/fig_main_results.pdf")
```
````

- No titles inside ggplot â€” use `fig-cap` chunk option instead
- Panel labels ("Panel A: ...") inside multi-panel figures are fine
- PDF format for figures (vector graphics); PNG only for raster content

## Title Page and Metadata

Set in the `.qmd` YAML header (overrides `_quarto.yml` defaults):

```yaml
title: "Paper Title"
author:
  - name: "Author Name"
    email: "email@university.edu"
    affiliation:
      - primary
abstract: |
  Abstract text here (150 words or fewer).
date: today
```

JEL codes and keywords go in the body after the abstract, as a raw LaTeX block:

````markdown
```{=latex}
\vspace{1em}
\noindent \textbf{JEL Codes:} X00, Y00

\vspace{0.5em}
\noindent \textbf{Keywords:} keyword one, keyword two
```
````

## Section Structure

Standard economics paper order:

1. Introduction
2. Background / Institutional Setting (if needed)
3. Data
4. Empirical Strategy
5. Results
6. Robustness
7. Conclusion

Sections use standard markdown headings: `## Introduction {#sec-intro}`. The `{#sec-intro}` anchor enables `@sec-intro` cross-references.

## Key Design Decisions

| Choice | Standard | Rationale |
|--------|----------|-----------|
| Quarto citeproc + CSL | Required | Native Quarto citation system; no manual LaTeX packages needed |
| Quarto cross-refs (`@fig-`, `@tbl-`, `@sec-`) | Required | Native syntax; Quarto injects hyperref automatically for PDF |
| `pdf-engine: xelatex` | Required | Better Unicode and font support |
| booktabs rules | Required | `\toprule`/`\midrule`/`\bottomrule`; never `\hline` |
| `threeparttable` or tabularray notes | Required | Every table must have notes |
| No figure titles in ggplot | Required | Titles go in `fig-cap` chunk option |
| `theme_minimal(base_family = "serif")` | Default | Paper figure theme; override only with explicit justification |
| Double-spacing always on | Required | `\doublespacing` in `_quarto.yml`; disable per-file only for handouts |
| 3 decimal places for coefficients | Default | Comma thousands separator for counts; matches etable/modelsummary defaults |

## What the Writer-Critic Checks

**Required (blocking deductions):**
- Manual `\ref{}` instead of `@fig-`/`@tbl-`/`@sec-` syntax (-3 per instance, max -10)
- `biblatex` or `natbib` loaded manually in a raw LaTeX block (-5)
- `\hline` instead of booktabs rules (-3)
- Missing table notes on any table (-5)
- Missing figure caption (`fig-cap`) on any figure (-5)
- Abstract over 150 words (-5)
- Missing JEL codes or keywords (-5)
- Notation inconsistency across sections (-3 per instance, max -10)
- Causal language in a descriptive paper (-5)
- Numbers in text that don't match script output (-10)

**Recommended (advisory â€” reported but not deducted):**
- Missing `{#sec-name}` anchors on sections (prevents cross-referencing)
- Raw variable names in table or figure labels
- Absolute paths in `\input{}` blocks
