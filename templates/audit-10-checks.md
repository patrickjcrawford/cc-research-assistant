# Replication Audit: 10 Checks

Verifier checklist for `/submit audit`. Each check is PASS or FAIL.
A single FAIL blocks submission. Document evidence for each check.

---

## Check 1: Master Script Exists and Runs

**Test:** `Rscript paper/replication/code/master.R` (or equivalent) completes without error.
**Evidence:** Terminal output, exit code 0.
**Common failures:** Missing packages, hardcoded paths, missing raw data files.

---

## Check 2: All Tables Reproduce

**Test:** Compare `output/tables/*.tex` against paper tables line by line.
**Tolerance:** Exact match for integers and strings; floating point within 0.001 for estimates.
**Evidence:** List each table file and PASS/FAIL.
**Common failures:** Stale `.tex` file not regenerated by master script.

---

## Check 3: All Figures Reproduce

**Test:** Confirm `output/figures/` contains one file per paper figure, generated by master script.
**Tolerance:** Visual match (pixel-perfect not required; color/font differences acceptable).
**Evidence:** List each figure file and PASS/FAIL.
**Common failures:** Figure saved to wrong path, ggplot theme differences across R versions.

---

## Check 4: README Is Complete

**Test:** README contains all required sections: Overview, Data Availability, Computational
Requirements, File Structure, Instructions, Table/Figure Crosswalk.
**Evidence:** Section-by-section checklist.
**Common failures:** Crosswalk missing rows, runtime not estimated.

---

## Check 5: Data Documentation Present

**Test:** Every dataset listed in README has: source URL or citation, access instructions,
and variable descriptions (or reference to an external codebook).
**Evidence:** List each dataset and documentation status.
**Common failures:** Restricted-access data listed without access instructions.

---

## Check 6: Scripts Are Numbered in Order

**Test:** Script file names begin with two-digit prefixes (`01_`, `02_`, ...) matching the
order in master.R.
**Evidence:** `ls code/` output.
**Common failures:** Files renamed after master script written, order mismatch.

---

## Check 7: Dependencies Are Listed

**Test:** `renv.lock` exists (R) or `requirements.txt` / `environment.yml` exists (Python),
and versions match what master script loads.
**Evidence:** File exists; spot-check 3–5 package versions.
**Common failures:** `renv.lock` not committed, package list outdated.

---

## Check 8: Runtime Is Documented

**Test:** README states estimated runtime with hardware specification.
**Evidence:** Locate runtime statement in README.
**Common failures:** Runtime omitted entirely; stated runtime wildly underestimated.

---

## Check 9: Output Paths Match Paper References

**Test:** Every `\input{}` or `\includegraphics{}` in `paper/main.qmd` resolves to a file
that the master script produces.
**Evidence:** Grep `\input` and `\includegraphics` in paper; verify each path.
**Common failures:** Paper references `paper/tables/tab1.tex` but script writes to `output/tables/tab1.tex`.

---

## Check 10: No Hardcoded Absolute Paths

**Test:** Grep all scripts for `/Users/`, `/home/`, `C:\Users\`, `C:/Users/`, `setwd()`, `os.chdir()`.
Zero matches required.
**Evidence:** Grep output (empty = PASS).
**Common failures:** Forgotten `setwd()` at top of script, path left in from development.
